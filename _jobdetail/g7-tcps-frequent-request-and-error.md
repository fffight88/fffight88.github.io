# G7

## 상태값 변경 (공통)

![](/assets/img/jobdetail/status_change.png)

1. 작업표시줄에 돌고래를 클릭하면 `sqlyog`가 열린다.
2. `g7_mariadb` db를 선택하고 `데이터조회` 탭을 선택
3. 보통 주문번호 또는 택스번호를 주면서 요청해온다. 주문번호면 `-- 3. 오더번호로 order_serial 찾기`에서, 택스번호면 `-- 2. tax_no로 order_serial 찾기`에서 주문시리얼을 검색한다(주문번호 또는 택스번호를 요청받은 것으로 변경하고 `F8`을 눌러 실행).
4. 주문상태값 변경이면 [주문 및 작업 상태값변경](#주문-및-작업-상태값변경), 세금계산서 상태값 변경이면 [세금계산서 상태값변경](#세금계산서-상태값변경)

<br>

### 주문 및 작업 상태값변경

> TCPS에서 발송완료 처리되었는데 G7은 여전히 주문접수 또는 주문전송 상태라 매출확정을 할 수가 없다.

이 케이스는 주문테이블과 작업테이블에서 해당 주문 및 주문에 속한 작업의 상태값을 모두 발송완료로 변경해주면 된다.

![](/assets/img/jobdetail/status_change2.png)

1. `-- 주문 / 결제 정보 --` 에서 첫번째줄 실행하여 해당 주문 조회, 상태값 부분을 찾아 `발송완료`로 바꾼다.
2. 네번째줄 실행하면 한 줄이 있을 터인데 체크>복사하여 한 줄을 생성한 후 `before_status`에 `주문전송`, `after_status`에 `발송완료` 해 주고 `created`는 클릭하여 달력이 띄워지면 `now`버튼을 눌러 현재날짜를 입력해준다. 그 후 디스크 버튼을 눌러 반드시 저장해준다.
    ![](/assets/img/jobdetail/status_change3.png)
3. 이제 `작업 정보`에서 첫번째 줄을 실행해 해당 작업을 검색
4. 두번째 줄을 실행, 위 1번과 같이 수정한다(작업이 여러개일 경우 전부 수정).
5. 네번재 줄을 실행, 위 2번과 같이 수정한다(작업이 여러개일 경우 전부 수정).
6. 위 2번 또는 5번 작업을 할 때 한 줄도 없어서 복사를 할 수 없는 경우가 있으면 2번 또는 5번은 생략한다.

<br>

> 주문담당자 변경이 안 된다. 주문담당자를 누구누구로 변경해 주세요.

1. G7 접속 후 `주문관리>주문목록` 화면에서 요청받은 주문번호로 검색하여 주문제목을 눌러 `주문상세보기` 화면
2. 첫번째 줄의 `접수자`의 드롭박스를 눌러 요청받은 이름으로 변경
3. 하단의 `수정` 눌러 업데이트
4. 이름을 변경하려고 할 때 오류메세지가 나오면서 변경이 안되면 지금 바꿀 수 없고 백두산매니저 오면 바꿔주겠다고 하기

<br>

> 매출확정된 주문인데 상태값이 주문전송, 발송완료 등으로 되어있어서 통계가 맞지 않는다.

상기 [TCPS에서 발송완료 처리되었는데 G7은 여전히 주문접수 또는 주문전송 상태라 매출확정을 할 수가 없다.](#주문-및-작업-상태값변경) 케이스와 동일하게 진행하는데 `발송완료` 대신 `매출확정`으로 바꿔준다. 이 케이스는 대부분 주문테이블에는 `매출확정`으로 제대로 되어있지만 작업테이블에서 일부 주문이 `매출확정` 상태로 업데이트가 안되어서 발생함.

### 세금계산서 상태값변경

#### 전송오류

1. MySQL에서 `-- 세금관련 정보 --` 의 첫번째 명령을 실행해 데이터가 전송오류 상태값으로 되어있는지 확인
2. 두번째 명령을 실행, `CMP_TAX_ST` 컬럼이 1 또는 9, `ERR_MSG` 컬럼에 에러메세지가 떠 있으면 전송오류인 게 맞다. `CMP_TAX_ST` 컬럼 값을 0으로 바꾸고 디스크버튼을 눌러 저장, 새로고침 버튼을 눌러 새로고침했을 때 `ERR_MSG`에 아무런 메세지가 없고 `CMP_TAX_ST`가 1, `FG_FINAL`이 1, `NO_TAX`에 값이 들어와있으면 정상처리 된 것
    ![](/assets/img/jobdetail/status_change4.png)
    ![](/assets/img/jobdetail/status_change5.png)
   
3. `-- 전송오류 후 재전송하여 성공한 경우 bo_tax update 하기--`의 첫번째 두번째 세번째를 순서대로 실행하여 `hometax_no`에 값이 들어와있고 `tax_status`가 `전송`으로 되어있으면 완료

<br>

#### 발행오류

> 세금계산서를 전송했는데 발행오류로 표시되는 경우

위 [전송오류](#전송오류)와 같은 방식으로 수정하면 됨

<br>

> 나중에 전송하려고 저장해놓은 세금계산서가 발행오류가 되어있어서 세금계산서 발행을 할 수가 없어요

1. MySQL에서 `-- 세금관련 정보 --` 의 첫번째 명령을 실행해 데이터 검색
2. 상태값을 `저장`으로 변경 후 세금계산서 발행 해보라고 안내
3. 발행 되면 ok, 발행안된다 하면 삭제하고 다시 하라고 하기
4. 원인은 현재 알지 못하고 대책으로 개발된 부분이 아직 없음

<br> 

#### 택스번호가 'T'로 시작하지 않고 'F'로 시작하는 경우

세금계산서가 선매출로부터 발행된 것일 때 F가 붙는데, MySQL에서 `-- 세금관련 정보 (선매출) --`의 첫번재 명령에 택스번호를 넣고 실행한 후 두번째와 세번째 명령어로 세금관련 테이블을 조회할 수 있다.

<br>

## 매출중복오류 수정

1. MySQL에서 `-- 3. 오더번호로 order_serial 찾기`에 요청받은 주문번호를 입력 후 실행하여 주문시리얼 검색
2. `-- 주문 / 결제 정보 --`의 두번째 줄을 실행했을 때, 하나의 주문에 대한 매출은 하나이므로 한 줄만 나와야 정상인데 두 줄이 나오는 경우가 있다. 이 때는 `serial` 번호가 더 큰 데이터를 체크박스눌러 선택하여 휴지통 버튼을 눌러 삭제

<br>

## 세금계산서 오류 상담 번호

`1688-7001` 전화해서 더존아이큐브 사용하는데 세금계산서 발행 중 오류때문에 전화했다고 하고 사업자번호(타라티피에스128-81-20943,타라그래픽스603-85-27237) 알려준 후 진행

# TCPS

## 주문 취소(=삭제) 처리

![](/assets/img/jobdetail/order_cancel.png)

1. 좌측 gnb 메뉴에서 `주문관리>주문목록` 오픈
2. `검색어` 란에 요청받은 검색어(주로 주문번호)를 입력하여 검색
3. 해당 주문의 `진행상황` 컬럼의 체크박스를 체크하고 드롭박스에서 `주문취소` 선택
4. 취소사유를 묻는 팝업이 뜨면 '담당자 요청' 정도로 기재하고 진행
5. 해당 주문의 주문제목에 적색 취소선이 그어진 것을 확인하고 요청자에게 통보

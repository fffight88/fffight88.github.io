# [G7 개발 및 유지보수 업무](http://g7.taragrp.co.kr)

![](/assets/img/portfolio/g7-1.png)

<br>

영업사원이 고객사로부터 주문을 받아 제품 발주에서부터 결제, 이후 세금 관련 업무와 매출조회 및 통계를 확인할 수 있는 타라그래픽스의 영업 및 회계업무용 ERP

<br>

## 개발 환경

* 운영체제      : Windows 11
* 플랫폼        : Web
* 웹서버        : AWS EC2(CentOS)
* DB서버        : AWS RDS
* 로컬서버      : Apache 2.4(XAMPP)
* 개발언어      : PHP 5.6, JavaScript(jQuery, Ajax)
* 프레임워크    : CodeIgnitor 3.0
* IDE           : Sublime Text, VS Code
* RDBS          : MySQL
* DBMS          : SQLyog
* 버전관리      : Jenkins, GitLab

<br>

## 프로젝트 - 외상거래 매출확정방식 변경

### 개요

* 기간 : 2022년 11월 하순 ~ 2023년 1월 중순
* 주목적 : 외상거래매출을 표현하는 방식 및 그 데이터가 실제 이루어지는 외상거래의 모습과 일치하지 않으므로, 당해연도(2022) 내에 문제되는 부분을 개선하라는 세무당국의 권고에 따라 프로젝트 진행
* 역할 : ( 약간의 )프론트엔드, 백엔드
* 프로젝트 기여도 : 60%

<br>

### 프로젝트 수행 배경

* 스마트팩토리 개발자과정 이수 후 첫 번째 직장으로, 그간 배우고 익혔던 지식 및 기술과는 거리가 먼 업무를 맡게 된 부분과, 팀에서 하나뿐인 개발자라는 사실에서 오는 부담감과 압박감이 컸음

<br>

### 어려웠던 부분

* 모든 부분
* PHP, 자바스크립트, jQuery, Ajax 등 모든 게 처음이나 다름없는 상태이므로 러닝커브 기간을 충분히 갖고 싶었으나 연말까지 프로젝트를 완료해야 하는 상황이라 주먹구구식으로 작업하게 됨
* 인쇄/디자인 및 회계업무에 대한 도메인 지식이 없는 상태에서 입사 후 일주일만에 수행하게 된 프로젝트인 탓에 실제 업무의 흐름을 모른 채로 요청하는 부분만 개발 및 개선작업을 진행하다보니 추후 추가로 손볼 부분이 매우 많았음
* 매출의 대부분을 차지하는 외상매출의 처리 로직을 손보는 작업이라 손봐야 할 부분이 적지 않았고, 그런 만큼 신입에게는 복잡하고 어려운 로직을 여러번 마주치며 어려움을 겪었음

<br>

### 프로젝트를 통해 배운 점, 성장한 면

* 수료했던 교육과정 중 웹의 비중은 매우 적어 웹에 대한 지식이나 경험은 거의 없었는데 프로젝트 수행을 통해 MVC 모델 기반의 웹애플리케이션 개발에 대해 많은 것을 알게 되고 체험해 봄
* 작업을 모두 마치고 배포 버튼을 누르는 순간의 긴장감과 배포 직후 쏟아져 나오는 에러들을 통해 개발자로서 가져야 할 마음가짐을 느끼게 됨
* 끌려가듯 수행한 프로젝트라 설계부터의 문서 작업이 제대로 이루어지지 못한 부분이 아쉬웠음

<br>
